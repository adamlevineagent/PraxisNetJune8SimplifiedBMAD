# Praxis Network - MVP Sprint Plan

This folder contains the complete, actionable development plan for the Praxis Network MVP.

## Methodology: Proving Grounds

Instead of traditional story points and velocity, this sprint is structured around a series of five **Proving Grounds**.

A Proving Ground is a demonstrable, integrated capability. To complete each Proving Ground, the development team must complete the series of tasks and stories within its corresponding folder and then successfully pass the "Checkpoint Challenge" defined in the `PROVING_GROUND_X.md` file.

This ensures we have a working, testable, end-to-end system at every stage of the build.

## Structure

- Each `XX_EPIC_...` folder contains the detailed stories for a major functional area.
- Each `STORY_...` file contains the "why" (the user story) and the "what" (the developer tasks).
- The `PROVING_GROUND_...` file in each folder contains the "test" that proves the epic is complete.
- `WORKLOG.md` is the master checklist for tracking progress.
